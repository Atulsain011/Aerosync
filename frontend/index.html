AeroShare User Guide

🚀 Welcome to AeroShare OS!

AeroShare OS is a multi-themed desktop web environment designed for sharing files at lightning speeds. It supports two primary transfer systems.

1. Server-Relay File Storage

Use this when you want to save a file permanently on the server's drive so others can download it at any time.

  • Upload: Simply drag & drop any file directly onto the Desktop canvas, or click Upload File in the Explorer window. You will immediately see the file appear in the file manager grid as [Uploading...] with real-time percentage indicators.
  • Resumability: Large files are uploaded in chunks. If your connection drops, the system automatically recovers the state and resumes from where it left off!
  • Download: Access the file explorer and click the download icon. It supports partial seek range streams (essential for video pause/seek).
2. Direct WebRTC P2P Transfer

Use this to send files directly between two devices (e.g., your laptop and your phone) without writing anything to the server's disk.

  • Discovery: Open the server URL (e.g., http://192.168.1.56:5000) on a second device. Both devices will blink and appear on each other's Network Peers Radar screen.
  • Transfer: Tap/click a peer node on the radar (or double-click), pick your file, and send. The recipient will see an accept invitation panel.
  • Efficiency: Files stream directly using WebRTC data channels in 16KB binary chunks with low-latency flow control, bypassing server storage entirely.
3. Theme & Connection Tweaks

Open the System Control panel to switch between Aero, Terminal, Classic macOS, and Sunset Vaporwave themes. You can also customize speed throttles, connection ports, and your user nickname.

4. Troubleshooting Personal Hotspots & Connection Issues

If you are using a personal hotspot (e.g. sharing from a mobile phone) and devices cannot connect or transfer files, please check the following solutions:

  • Linux Firewall Blocking: Linux firewalls often block incoming connections on port 5000. Open a terminal on the host laptop and run: sudo ufw allow 5000/tcp.
  • Android/iOS Hotspot Isolation: By default, a phone hosting a hotspot cannot establish connections to its own client devices. To resolve this:
    • Turn on the Wi-Fi Hotspot on the laptop instead, and connect the phone to the laptop's network.
    • Or use USB Tethering (connect phone via USB and enable USB tethering).
    • Or connect both the laptop and phone to a third device's hotspot or local Wi-Fi router.
  • WebRTC Direct Transfer Blocked: Mobile hotspots often block direct peer-to-peer UDP streams (AP isolation). If direct P2P transfer hangs at "Connecting...", don't worry! Simply upload the file to My Shared Files (Server-Relay). Connected devices can download it directly from the explorer list via HTTP.
